GPT做的唯一事情，就是文字接龙；给它一个不完整的句子，它去完成后续。
LLM如何回答用户问题呢？就是接着用户问题，继续接龙。
给每一个可选择的符号（token）/词 一个概率分布。——每次回答同一个问题，几率不不一样。

每次选择几率最大的token，得到的回答不一定最好。

每个LLM设置的token 是不同的。
token 必须是可穷举的，才能可以给每个token一个概率分布；
每天都在创建新的英文单词，不能直接作为token。
```
un
kill
able
unkillable 一个英文单词，可能拆分成3个token。
```

多轮对话：把过去对话的内容，通通加入当前对话的prompt里。


LLM：类神经网络，Transform
y=Ax+B
输入一个不完整句子，输出一个概率分布。
LLM上亿个参数。

pre-train预训练，从网络爬取数据、直接学习，无人类督导；看过大量资料后 直接让它文字接龙，但有时会任意接龙，因为它不知道人类期望它回答、做什么。
InstructGPT 督导式学习，有人类督导，告诉LLM期望你做什么。

有了pre-train，督导式学习只需少量指导。——画龙点睛
有了指导，预训练很差的模型，都能胜过预训练很强的模型。

增强式学习RL
人类老师不再直接督导（完整的正确答案），而是提供反馈。——RHLF
同一个问题，第二次回复时会询问，我这次回答有没有比上一次好；

增强式学习，放在训练的最后一段；模型要有一定的能力，偶有佳作，用户反馈后，模型调整让好的回复提高几率。
这样模型只能把某一个问题回复好，换个场景呢。所以GPT的增强式学习分两个阶段
- 先学习人类偏好 reward model
- 获得反馈。



督导式学习+增强式学习，现在也叫alignment，向人类偏好对齐。
